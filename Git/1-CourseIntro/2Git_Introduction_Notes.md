# Git & GitHub Course Notes

## Introduction to Git

Git is the world's most popular version control system.

A version control system (VCS) is software that tracks and manages changes to files over time.

Version control systems allow users to:
- Restore earlier versions of files
- Compare changes between versions
- Undo changes
- Manage file history

## Git is One VCS

Git is one of many version control systems available today.

Other well-known VCS tools include:
- Subversion (SVN)
- CVS
- Mercurial

They share similar goals but differ in how they work.

Git is widely adopted and is the preferred version control system for many developers.

## What is Git?

Git is a free and open-source distributed version control system designed to handle projects of all sizes with speed and efficiency.

Git provides:
- Easy learning curve
- Small footprint
- Fast performance
- Local branching
- Staging areas
- Multiple workflows

It improved many areas compared to older SCM tools like:
- Subversion
- CVS
- Perforce

## History of Git

Linus Torvalds is the creator and main developer behind Linux and Git.

In 2005, while working on Linux, he became frustrated with existing version control systems because they were slow, closed-source, and often paid.

He created Git as a "stupid content tracker" during development.

The name Git was later used and is commonly explained as:
- Global Information Tracker

## Git vs GitHub

### Git

Git is version control software that runs locally on your computer.

You:
- Do not need an account
- Do not need internet access
- Can use Git without GitHub

### GitHub

GitHub is a cloud service that hosts Git repositories and makes collaboration easier.

GitHub allows developers to:
- Store repositories online
- Share projects
- Collaborate with teams
- Manage code remotely
